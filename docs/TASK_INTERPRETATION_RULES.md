# Правила интерпретации задач агентом

## Основное правило

Многострочный пользовательский ввод должен восприниматься как единое техническое задание.

## Запрещено

- Разбивать строки на отдельные команды.
- Искать отдельно слова:
  - title
  - price
  - rating
  - availability
- Подменять сайт другим.
- Выполнять поиск на Ozon, если пользователь указал другой URL.
- Интерпретировать список полей как независимые запросы.

## Обязательно

- Определять тип задачи перед выполнением.
- Если указан URL — работать только с ним.
- Понимать, что списки после слов «Нужно собрать» являются схемой данных.
- Перед запуском кратко формулировать понимание задачи.
